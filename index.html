Why This Exists

Most Mac gamers open an emulator, dig through folders and hunt for ROM files every single time they want to play. roms2app.sh creates lightweight native launchers so your games show up directly in Launchpad, Spotlight and Finder — just like any Mac app.

Safe Re-Runs

External Drives

Designed for large libraries stored on external drives — nothing is copied or moved.

Recommended Emulators

To play PS2 and Switch games on a Mac, you first need an emulator — software that mimics the original hardware so your Mac can run the games. roms2app.sh creates the launchers, but the emulators are what actually run the games. Here's what to install:

PCSX2 — PlayStation 2

The gold standard for PS2 emulation. Free, open-source, and actively maintained. On Apple Silicon Macs it runs the vast majority of the PS2 library at full speed. Download the macOS build from the official site.

After installing PCSX2, open it at least once and configure your BIOS before using roms2app.sh. The BIOS is a small file extracted from a real PS2 console — it is required for emulation to work.

Astris — Nintendo Switch

A newer Switch emulator for macOS with strong Apple Silicon performance. Still in active development but already capable of running many popular titles. Download from the GitHub releases page.

Switch emulation also requires keys and firmware dumped from a real Nintendo Switch console. Astris will guide you through the setup on first launch.

Getting Started

Follow these steps in order. If this is your first time doing emulation on Mac, read each note carefully.

Step 1 — Install the emulators

Download and install PCSX2 for PS2 and/or Astris for Switch. Launch each one at least once and complete the initial setup (BIOS for PCSX2, keys + firmware for Astris).

Step 2 — Organise your game files

Store your game files (ROMs) on an external drive or in a dedicated folder. The script expects PS2 games in a /ps2 folder and Switch games in /nswitch. Supported formats: .iso for PS2, .nsp / .xci for Switch.

Only use game files from discs you personally own. Downloading ROMs for games you don't own is illegal.

Step 3 — Configure the script

Open roms2app.sh in any text editor and adjust the paths at the top of the file to match your setup:

SOURCE_DIR_PS2="/Volumes/Games/ps2"
SOURCE_DIR_SWITCH="/Volumes/Games/nswitch"
DEST_DIR="/Applications/Games"

Replace the example paths with the actual locations on your Mac. Drive names and folder names are case-sensitive.

Step 4 — Run the script

Open Terminal (Applications → Utilities, or Spotlight) and run:

chmod +x roms2app.sh
./roms2app.sh

The script will scan your game folders and create a native .app launcher for each game inside /Applications/Games. Your games will appear in Launchpad immediately.

You only need to run the script again when you add new games. Existing launchers are skipped automatically.

Step 5 — App icons

After the script finishes, your game launchers will display a generic white icon. Automatic icon fetching is not yet part of the script — it's coming in a future update. Until then, you can apply custom icons manually in seconds:

  1. Go to macosicons.com and search for your game title.
  2. Download the .icns file.
  3. In Finder, right-click the .app inside /Applications/Games and choose Get Info.
  4. Drag the .icns file onto the small icon thumbnail in the top-left corner of the Get Info window.
  5. Done — the icon updates immediately in Launchpad and Finder.
Automatic icon assignment is planned for a future release — the script will handle this step entirely on its own.

FAQ

Do I need to run the script every time I want to play?
No. Only run it again when you add new games to your library.

Does the script copy or move my game files?
No. Your ROMs stay exactly where they are. The script only creates small launcher files.

Can I search for games via Spotlight?
Yes. The launchers behave like regular macOS apps and are fully indexed by Spotlight.

My external drive isn't always connected — will the launchers break?
The launchers will show an error if the drive isn't mounted. Just connect the drive first and try again.
